Проект "Система управления продуктовыми командами".


1. Сущности: Пользователи, Роли, Задачи, Продуктв, Информационные системы, Типы задач, Спринты, комментарии к задачам
2. Spring Security авторизация доступ только для авторизованных пользователей, ACL права на уровне продуктор, применяются для продуктов, задач, спринтов.
Обнаследовал CustomMutableAclService extends JdbcMutableAclService и CustomAclPermissionCacheOptimizer extends AclPermissionCacheOptimizer доработал методы для разаление прав по атрибуду продукт.
3. Конверторы данных в из Dto реализованы через mapStruct
4. Для меню использовал Themleaf fragment
4. Добавил обновление установку прав при администрировании пользователе, т.е в карточки модно дость доступ пользователю к продукту и данные занесутся в таблицы ACL
Для всех продуктов есть доступ у роли ROLE_ADMIN
5. Реализовал интерфейс для всех справлчников и задач.
6. реализовал каширование на уровне сервиса Информационных систем, типов задач и пользователей,  с очисткой кэша при изменениях.
7. Реализовал загрузка задач из файла Excel через Spring Batch. Пример загрузочного файла лежит папке ресурсов taskfile.xlsx
8. Реализовал отправку WebHook в смежный трекер при соответсвующей настроки в Инфомационной системе. Используются поля useWebHook и urlWebHook.
 Отправка реализована через RestClient
9. реализована отправка писем об изменении задачи ответственному по задаче через Spring Mail.
10. Вызов  WebHook и отправка посты реализована через Spring Integration
11.Реализовано автоматическа регистрация задач по сообщениям топика Kafka (подлразумеваются специалные топики с ошибками, если принимающа система не может загрузить сообщение)
 Топика настраиваетсыя в application.yml . Пришлось для таких задач реализовать установку аудетификации в Spring context под системной учеткой SYSTEM.
12. ИСпользуется Spring-validation для валидавиивходящих данных.
13. Реализованы некоторые  UNIT-тесты
